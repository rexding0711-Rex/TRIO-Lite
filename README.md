# 🎭 TRIO 基础版

> **日常办公的 AI 搭档——三个视角，一个答案。**

你是不是也有这种感觉：问 AI 一个问题，它答得飞快、语气自信——但你就是有点不放心？

TRIO 就是为这个设计的。**同一个 AI，换三种语气问同一件事：**

- 🎨 **好奇宝宝** 先到处看看——"诶，这个领域好有意思，我发现了这些……"
- 🔍 **较真鬼** 冷眼扫一遍——"等一下，你这三个地方都没证据"
- 🔧 **实干家** 综合两边的发现——"好了，结论是这样，下一步做这个"

三个小伙伴聊完一轮，你拿到的不是"AI 说对就对"——是**互相挑过刺的答案**。

**不是换模型，是换了三种问法。**

---

## 🎮 5 分钟——从零到出发

```bash
# 1. 装 Claude Code（如果还没有）
npm install -g @anthropic-ai/claude-code

# 2. 下载 TRIO
git clone https://github.com/rexding0711-Rex/TRIO-Lite.git
cd TRIO-Lite

# 3. 检查背包——看看还缺什么装备
bash setup.sh

# 4. 创建角色——TRIO 想认识你
bash init.sh
```

init.sh 会跟你聊四个话题：你做什么、东西放哪、在意什么标准、要不要让 TRIO 越用越懂你。2 分钟。聊完之后，你的 TRIO 就不是默认设置了——它知道你在意什么。

```bash
# 5. 出发！
claude
```

进入后试试：`/速判 <你想了解的东西>`

卡在环境那步了？→ 📖 [新手村指南 (SETUP.md)](SETUP.md)

## 🎯 适合你，如果你

- 经常要做「没有标准答案」的判断（尽调、竞品、技术选型）
- 觉得单个 AI 给的答案「太流畅了，有点不放心」
- 想要几个不同视角帮你互相挑刺、拿到更靠谱的结论

## 🙅 可能不适合你，如果你

- 只需要查事实（搜索引擎更快）
- 简单问答就够了，不需要交叉验证
- 还没用过任何一个 AI 工具——建议先玩熟 Claude Code 再来

## ⚔️ 一个真实的小故事

对一家 AI 3D 公司的尽调（2026 年 6 月）——同一个 API，三个小伙伴轮流上：

- 🎨 **好奇宝宝** 先探路：技术领先，HackerNoon 盲测第一，ARR $12M
- 🔍 **较真鬼** 逐条验证：用户数口径有水分，ARR 过期 10 个月未更新，"世界模型"只是研究 Demo
- 🔧 **实干家** 拍板：不是骗局（6/7 可信度），但叙事水分显著 → **建议观望至下一份 ARR 数据公布**

同一个模型，三种问法——单独问说"技术领先"，三个小伙伴聊完说"水分显著，等数据"。

## 🎭 三个小伙伴

同一个 API，换了三种语气——每个小伙伴有一种性格：

| 命令 | 小伙伴 | 擅长 |
|------|------|------|
| `/doer` | 🔧 实干家 | 动手做——"这事怎么搞定" |
| `/auditor` | 🔍 较真鬼 | 挑刺——"哪有漏洞" |
| `/suggestor` | 🎨 好奇宝宝 | 发散——"还能怎么看" |
| `/all` | 🎭 三合一 | 三个小伙伴一起上 |

## 🗺️ 日常办公能做什么

| 命令 | 做什么 | 适合场景 |
|------|--------|---------|
| `/速判 <目标>` | 快速评估，10-15分钟 | 想快速了解一个公司/话题 |
| `/尽调 <目标>` | 深度调研（含逆向工程、因果分析） | 需要靠谱结论的重要判断 |
| `/竞品 <目标>` | 竞品分析 | 了解赛道和对手 |
| `/内容 <主题>` | 写文档/报告/创作 | 出 Word、PPT、Markdown |
| `/复盘 <项目>` | 项目审计复盘 | 做完一件事后总结经验 |

## 📁 目录结构

```
TRIO基础版/
├── .claude/commands/       ← 斜杠命令（~15个）
├── config/
│   ├── roles/              ← 角色定义
│   ├── scenarios/          ← 场景 SOP
│   └── protocols/          ← 协作协议
├── scripts/                ← 管理脚本
├── knowledge/              ← TRIO 自身知识
├── docs/                   ← 架构文档
├── setup.sh                ← 背包检查
├── init.sh                 ← 新用户引导
└── state.json              ← 运行时状态

你的数据在 $TRIO_DB/:
├── 项目/     ← 工作产出
├── 知识库/   ← 积累的资料
└── 归档/     ← 备份
```
