核心机制：三视角协作

TRIO 不是三个 AI 模型。它是同一个 AI，换三种思考方式：

面具性格擅长一句话
🎨 好奇宝宝发散 · 洞察 · 联想行业扫描、竞品情报、跨领域类比「诶，这个方向有点意思……」
🔍 较真鬼质疑 · 审计 · 验证事实核查、成本拆解、逻辑漏洞挖掘「等一下，你这句话没有证据」
🔧 实干家收敛 · 决策 · 交付架构设计、方案选择、文件打包「好了，结论是这样，下一步做这个」

三个视角各自独立输出 → 实干家综合 → 一份「已经被挑过刺」的结论。

不是换模型，是换了三种问法。

5 个场景 SOP

每个场景是一套标准化的多步骤流程，不是简单问答：

场景步骤流程输出
尽调7 步任务编排 → 行业定位 → 事实核查 → 成本拆解 → 叙事解剖 → 综合评分 → 打包评分卡 + PDF
竞品分析5 步扫描 → 对比 → 护城河 → 威胁 → 策略竞品矩阵 + 雷达图
审计复盘5 步目标 → 结果对比 → 偏差 → 根因 → 改进审计报告
内容交付4 步需求 → 大纲 → 初稿 → 审核 → 终稿文章 / 方案 / 报告
快速评估3 步扫描 → 核查 → 判定评级 + 一句话结论

技术架构

用户 → 15 个命令 → 场景路由（5 SOP）→ 角色调度（11 角色）→ 三视角输出 → 基础设施（备份 / 同步 / 进化 / 门禁）

适合你 / 不适合你

✅ 适合你，如果你

  • 觉得 AI 回答太快太自信，想要被验证过的答案
  • 做分析 / 调研 / 评估类工作，需要结构化输出
  • 想看「三个角度讨论同一件事」会不会发现盲区
  • 对 AI 工作流感兴趣，想有一套开箱即用的框架

❌ 不适合你，如果你

  • 只需要简单问答（ChatGPT 就够了）
  • 不需要结构化分析（TRIO 有学习成本）
  • 对命令行感到不适（TRIO 基于终端）

TRIO 不是一个更好的 ChatGPT。
它是你的思维搭档——三个性格不同的小伙伴，陪你一起想清楚。
