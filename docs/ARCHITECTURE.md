# TRIO 基础版 架构说明

> 三层模块化架构——聚焦日常办公流：文档写作、信息搜集、快速评估、竞品了解、项目复盘。

## 设计理念

TRIO 基础版将信息处理任务拆分为三个独立视角：

- **Suggestor（发散）** — 多源搜索、跨域联想、创意构思
- **Auditor（审计）** — 事实核查、逻辑校验、反事实检验
- **Doer（工程）** — 综合判定、代码实现、文件交付

三种面具在同一会话内按场景 SOP 接力协作。任何重要判断至少经过两个独立视角交叉验证。

## 三层架构

```
交互层  → .claude/commands/     (~15个命令，用户入口)
能力层  → config/roles/ + scenarios/ + protocols/  (引擎)
基础层  → scripts/ + mgmt.sh + knowledge/ + docs/  (基础设施)
```

- **交互层**：用户看到和使用的斜杠命令。覆盖日常办公高频场景——速判、尽调、竞品、内容、复盘。
- **能力层**：3个面具 + 5个场景SOP + 共享协议。场景驱动，面具接力。
- **基础层**：Shell脚本、知识库、状态管理、禁区守护。

层间通过 mgmt.sh 跨层通信。上层依赖下层，下层不知上层。

## 核心机制

- **管道协议** — 三面具协作的标准接口。发散→审计→收敛
- **自验证闭环** — 交付前自检 → 修正 → 再检
- **上下文隔离** — Step 间只传 ≤500 字结构化摘要
- **优雅降级** — 高风险场景不拒绝服务，降级到保守模式执行
