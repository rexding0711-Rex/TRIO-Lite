# TRIO 基础版 — 功能介绍

> 日常办公的 AI 搭档——三个视角，一个答案。
> 基础版 1.0 · 开源 · MIT

---

## 一句话

**同一个问题，三种性格的 AI 从不同角度审视——互相挑刺之后，你拿到的不再是「AI 说对就对」，而是经过对抗验证的答案。**

---

## 核心机制：三视角协作

TRIO 不是三个 AI 模型。它是**同一个人工智能，换三种思考方式**：

| 面具 | 性格 | 擅长 | 一句话 |
|:---:|---|------|------|
| 🎨 **好奇宝宝** | 发散·洞察·联想 | 行业扫描、竞品情报、跨领域类比 | 「诶，这个方向有点意思……」
| 🔍 **较真鬼** | 质疑·审计·验证 | 事实核查、成本拆解、逻辑漏洞挖掘 | 「等一下，你这句话没有证据」
| 🔧 **实干家** | 收敛·决策·交付 | 架构设计、方案选择、文件打包 | 「好了，结论是这样，下一步做这个」

三个视角各自独立输出 → 实干家综合 → 得到一份「已经被挑过刺」的结论。

**不是换模型，是换了三种问法。**

## 15 个命令

### 一键分析类

| 命令 | 功能 | 适用场景 |
|------|------|---------|
| `/速判 <目标>` | 快速评估（3-5分钟） | 想看一个东西靠不靠谱，不想等 |
| `/尽调 <目标>` | 全流程尽调（7步） | 深度评估公司/项目/方案 |
| `/竞品 <目标>` | 竞品分析 | 产品对比、市场定位 |
| `/内容 <主题>` | 内容创作交付 | 写文章、报告、方案 |
| `/复盘 <项目>` | 快速审计复盘 | 项目做完了，回头看看 |

### 自由模式类

| 命令 | 功能 |
|------|------|
| `/trio-run <描述>` | 自由模式——三视角同时输出，不限场景 |
| `/trio-resume <id>` | 恢复之前的 run |
| `/trio-abort <id>` | 取消正在进行的 run |
| `/日进化` | 每日知识沉淀——今天学了什么 |
| `/能力镜像` | 查看当前能力成长情况 |

### 单面具类

| 命令 | 功能 |
|------|------|
| `/doer` | 切换到实干家面具 |
| `/auditor` | 切换到较真鬼面具 |
| `/suggestor` | 切换到好奇宝宝面具 |

## 5 个场景 SOP

每个场景是一套标准化的多步骤流程，不是简单问答：

| 场景 | 步骤 | 输出 |
|------|:---:|------|
| **尽调** | 7步：任务编排→行业定位→事实核查→成本拆解→叙事解剖→综合评分→打包 | 评分卡 + 核查清单 + PDF |
| **竞品分析** | 5步：扫描→对比→护城河分析→威胁评估→策略建议 | 竞品矩阵 + 雷达图 |
| **审计复盘** | 5步：目标回顾→结果对比→偏差分析→根因挖掘→改进清单 | 审计报告 + 改进建议 |
| **内容交付** | 4步：需求分析→大纲→初稿→审核→终稿 | 文章/方案/报告 |
| **快速评估** | 3步：扫描→核查→判定 | 评级 + 一句话结论 |

## 11 个工程角色

每个角色是一个专业技能包，按需调用：

| 角色 | 职责 |
|------|------|
| 产业情报工程师 | 行业扫描、数据收集、竞品初筛 |
| 事实核查工程师 | 逐条验证声明、溯源、置信度标注 |
| 成本拆解工程师 | BOM 拆解、隐性成本、估值合理性 |
| 因果分析工程师 | 因果 DAG 构建、反事实检验 |
| 复盘工程师 | 系统性逻辑缺陷挖掘、PR vs 真相 |
| 产品架构师 | 10 维综合评分、最终判定、方案选择 |
| 任务编排工程师 | 复杂目标分解为子任务 DAG |
| 内容生产工程师 | 文章/报告/方案的撰写与打磨 |
| 全栈交付工程师 | PDF/Word/Markdown/HTML 打包 |
| 认知镜像工程师 | 能力追踪、偏差校准 |
| 资产盘点工程师 | 知识库索引维护、技能提取 |

## 技术架构

```
用户 → 15 个斜杠命令
       ↓
    场景路由 (5 个 SOP)
       ↓
    角色调度 (11 个工程角色)
       ↓
    三视角输出 (好奇→较真→实干)
       ↓
    基础设施 (备份/同步/进化/门禁)
```

- **开源协议**: MIT
- **依赖**: Claude Code（npm 全局安装）
- **数据**: 100% 本地，不上传

## 适合你，如果你

- 觉得 AI 回答太快太自信，想要「被挑过刺」的答案
- 做分析/调研/评估类工作，需要结构化输出
- 想看看「三个角度讨论同一件事」会不会发现盲区
- 对 AI 工作流感兴趣，想有一套开箱即用的框架

## 不适合你，如果你

- 只需要简单问答（ChatGPT 就够了）
- 不需要结构化分析（TRIO 有学习成本）
- 对命令行感到不适（TRIO 基于终端）

---

> TRIO 不是一个更好的 ChatGPT。它是你的思维搭档——三个性格不同的小伙伴，陪你一起想清楚。
