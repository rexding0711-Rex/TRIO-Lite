# 技能：平台生态型公司逆向工程

- 场景：竞品 / 尽调
- 来源：示例
- 置信度：★★★★★

## 方法思路
1. 找"标准接口"——这家公司有没有定义行业标准（如 CUDA 之于 GPU 计算）？定义了，护城河才真
2. 追网络效应——开发者 / 用户 / 合作伙伴的增长曲线
3. 量切换成本——离开这个平台要付多少代价（钱 / 时间 / 生态）
4. 看全栈集成度——覆盖了价值链多少个环节
5. 找"替代时刻"——有没有跨平台方案正在降低迁移门槛

## 踩过的坑
- 财务数据会掩盖竞争威胁——高增速让人觉得无敌，替代方案却在底层侵蚀
- "估值锚定"陷阱——90% 市场份额让人忽视"份额掉 5% 可能对应估值掉 50%"

## 可复用模式
- **标准接口 = 平台模式**：定义了行业标准 → 护城河真；没定义 → 只是大号 SaaS
- **三重锁定检测**：技术锁定 + 生态锁定 + 供给锁定，三个都占 = 真护城河；只占其一 = 脆弱
- **"超预期但盘后跌"反模式**：连续超预期业绩但股价盘后跌 → 市场在 price in 一个比业绩更大的风险
